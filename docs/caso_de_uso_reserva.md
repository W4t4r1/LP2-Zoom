# Especificación de Caso de Uso: CU-01

Este documento detalla la documentación formal del caso de uso **CU-01: Registrar solicitud de reserva de laboratorio** solicitado para la Evaluación de Programación y UML.

---

## Ficha del Caso de Uso: CU-01

| Campo | Detalle |
| :--- | :--- |
| **Código** | **CU-01** |
| **Nombre** | **Registrar solicitud de reserva de laboratorio** |
| **Actor principal** | **Docente** (Usuario solicitante que requiere el uso del laboratorio para sus sesiones de clase). |
| **Actores secundarios** | **Sistema** (encargado de las validaciones automáticas), **Administrador de Laboratorios** (recibe la notificación para aprobar/rechazar). |
| **Objetivo** | Permitir al docente registrar una solicitud de reserva de un laboratorio específico para una fecha y hora determinadas, indicando los recursos requeridos para el dictado de su clase. |
| **Precondición** | 1. El Docente debe haber iniciado sesión y estar autenticado en el sistema.<br>2. El laboratorio seleccionado debe estar registrado y habilitado en el catálogo.<br>3. La fecha y hora solicitadas deben encontrarse dentro del calendario y horario académico vigente. |
| **Postcondición** | 1. La solicitud de reserva queda registrada en la base de datos con estado `"Pendiente de Aprobación"`.<br>2. Se asocia un código único de solicitud a la reserva.<br>3. Se bloquea preventivamente el horario solicitado para evitar colisiones.<br>4. Se envía una notificación de confirmación de registro al Docente y un correo de aviso al Administrador de Laboratorios. |
| **Flujo principal** | **1.** El Docente ingresa al menú de reservas del sistema y selecciona la opción "Registrar solicitud de reserva".<br>**2.** El Sistema muestra el formulario de solicitud de reserva con los datos del docente precargados.<br>**3.** El Docente selecciona el laboratorio, la fecha, la hora de inicio y la hora de finalización.<br>**4.** El Docente ingresa la asignatura, el número estimado de estudiantes y selecciona los recursos adicionales requeridos (proyector, software específico, guías físicas, etc.).<br>**5.** El Docente presiona el botón "Enviar Solicitud".<br>**6.** El Sistema valida que todos los campos obligatorios estén completos y verifica en tiempo real que el laboratorio no tenga otra reserva confirmada en la fecha y rango de horas seleccionados.<br>**7.** El Sistema registra la solicitud en la base de datos con estado `"Pendiente"` y genera un código único de seguimiento (ej. `RES-2026-009`).<br>**8.** El Sistema muestra un mensaje de éxito en pantalla con el código generado y envía una notificación de confirmación al correo institucional del docente. |
| **Flujo alternativo** | **3a. Laboratorio no disponible en el horario seleccionado:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 6, el Sistema detecta que el laboratorio ya se encuentra reservado u ocupado (ej. por mantenimiento).<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El Sistema muestra una alerta de "Horario no disponible" y presenta alternativas de horarios disponibles para el mismo laboratorio o laboratorios con capacidad similar en el mismo día.<br>&nbsp;&nbsp;&nbsp;&nbsp;3. El Docente selecciona una alternativa sugerida y el flujo continúa en el paso 5 del flujo principal.<br><br>**5a. Cancelar el registro:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En cualquier momento previo a presionar "Enviar Solicitud", el Docente presiona "Cancelar".<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El Sistema descarta la información ingresada y redirige al Docente a la pantalla de inicio del portal de reservas (Fin del caso de uso). |
| **Reglas de negocio** | **RN-01 (Anticipación de Reserva):** Toda solicitud de reserva debe ser registrada con un mínimo de 48 horas de anticipación a la fecha solicitada.<br>**RN-02 (Aforo Máximo):** El número de estudiantes ingresado no debe superar la capacidad de estaciones de trabajo del laboratorio seleccionado.<br>**RN-03 (Duración del Bloque):** La duración mínima de una reserva es de 2 horas académicas y la máxima de 4 horas académicas.<br>**RN-04 (Límite de Peticiones Pendientes):** Un docente no puede registrar una solicitud si ya cuenta con más de 3 reservas en estado "Pendiente de Aprobación" simultáneamente. |
| **Datos de entrada** | - ID/Código de Docente (automático de la sesión).<br>- ID del Laboratorio seleccionado.<br>- Fecha de reserva (Formato: DD/MM/AAAA).<br>- Hora de inicio y hora de fin (Formato: HH:MM).<br>- Asignatura / Curso.<br>- Cantidad de estudiantes.<br>- Recursos opcionales requeridos. |
| **Datos de salida** | - Código de solicitud de reserva generado.<br>- Estado de la solicitud (`"Pendiente"`).<br>- Fecha y hora del registro de la solicitud.<br>- Correo electrónico de notificación enviado. |
| **Excepciones** | **EX-01: Falla de conexión a la Base de Datos:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 7, el sistema no logra establecer conexión con la base de datos Supabase o local.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. El Sistema interrumpe la transacción, muestra un mensaje informativo ("Servicio temporalmente no disponible, intente más tarde") y no guarda ningún cambio.<br><br>**EX-02: Cuenta inhabilitada por sanción:**<br>&nbsp;&nbsp;&nbsp;&nbsp;1. En el paso 6, el sistema verifica el estado del docente en el padrón.<br>&nbsp;&nbsp;&nbsp;&nbsp;2. Si la cuenta del docente tiene una sanción vigente (ej. por mal uso previo), el Sistema muestra una alerta indicando la inhabilitación y cancela el flujo de registro. |
